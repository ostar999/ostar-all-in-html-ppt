一、安装 Claude Code

Claude Code 是 Anthropic 官方出品的 CLI 编程助手。先装 Node.js 20/22 LTS，再一行命令搞定。

📋 前置条件

node -v  → Node.js ≥ 20
npm -v  → npm ≥ 10

$ npm install -g @anthropic-ai/claude-code

$ claude --version

📖 官方文档：docs.anthropic.com/en/docs/claude-code

step · 2 / 7

二、准备 DeepSeek API Key

登录 DeepSeek 开放平台，创建 API Key。这是后续 CC Switch 配置的核心凭证。

🔑 操作步骤

  1. 打开 platform.deepseek.com/api_keys
  2. 登录你的 DeepSeek 账号
  3. 点击「创建 API Key」
  4. 复制并安全保存 sk-xxxxxxxxxxxxxxxx
⚠️ Key 只显示一次，请妥善保存

step · 4 / 7

四、配置 DeepSeek Provider

在 CC Switch 中新建 Provider，填入 DeepSeek 的 Anthropic 兼容接口地址和 API Key。

⚡ 关键信息

DeepSeek 提供了与 Anthropic API 兼容的接口，Claude Code 可以直接使用，无需任何中间层。

cc-switch · provider config
# 新建 Provider 选择
Provider Type:  DeepSeek
# 或 Custom Provider

# 填写配置
Base URL:
  https://api.deepseek.com/anthropic

API Key:
  sk-xxxxxxxxxxxxxxxx

# ✅ Anthropic 兼容，Claude Code 直接可用

step · 5 / 7

五、配置模型槽位映射

Claude Code 有三个模型槽位，分别映射到 DeepSeek 模型：

Claude Code 槽位映射模型定位
Haiku → deepseek-v4-flash 快速·便宜
Sonnet → deepseek-v4-pro 主力推理
Opus → deepseek-v4-pro 复杂任务

💡 核心思路：Sonnet 和 Opus 都走 V4 Pro（强推理），Haiku 走 V4 Flash（快 + 省）。这是目前社区最主流的配置。

step · 6 / 7

六、推荐高级配置

🎯 主模型

deepseek-v4-pro[1m]

[1m] = 100 万 Token 上下文窗口。不加此标记时使用默认较短上下文。

🤖 子代理模型

让子 Agent 用 Flash，显著降低成本：

.zshrc / settings.json
# 环境变量方式
export CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash

# 或 settings.json
{
  "CLAUDE_CODE_SUBAGENT_MODEL":
    "deepseek-v4-flash"
}

💡 Pro 做规划推理，Flash 做子任务执行 = 性价比最优。

Q: 模型名后面的 [1m] 是什么？
开启 100 万 Token 上下文窗口。不加则使用默认较短上下文。适合分析大型代码库或多文件重构。
Q: CC Switch 本地代理模式能用 [1m] 吗？
部分教程指出本地代理模式下不要直接填写 [1m]，可能导致识别异常。请按 CC Switch 当前版本的说明配置。
Q: V4 Pro vs V4 Flash 怎么选？
Flash 快 + 便宜，日常编码用
Pro 强推理，复杂重构和架构设计用
社区推荐：主Agent=Pro + 子Agent=Flash
