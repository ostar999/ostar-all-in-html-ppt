💬 使用方法：如何与 AI 对话生成 PPT

💬

你不需要写代码——你只需要用自然语言描述你的需求

这个 Skill 安装后，AI 会自动识别你的 PPT 需求。关键是你怎么描述，决定了 AI 怎么帮你做。好的对话技巧能让 AI 一次就给你满意的结果，而不是反复修改。

触发关键词

当你的对话中包含这些词时，AI 会自动调用此 Skill：

PPTslidesdeck 幻灯片演讲稿分享稿 presentationpitch decktech sharing 小红书图文做一份帮我写个

⚠️ 如果你有安装其他制作 html、ppt 的 agent skill，为了不混淆，强烈建议你使用 /ostar-all-in-html-ppt 或者明确在对话里要求使用 ostar-all-in-html-ppt skill 制作。

对话模板：3 个信息一次给全

AI 在动手做 PPT 之前，需要知道 3 件事。一次性说清楚，效率最高：

①
内容和观众

做什么主题？几页？给谁看（工程师 / 投资人 / 小红书读者 / 学生）？

②
风格偏好

喜欢什么风格？（暗色科技 / 极简白 / 小红书风 / VC路演风）不确定可以让 AI 推荐。

③
模板选择

用哪个现成模板打底？（技术分享 / 周报 / 产品发布 / 融资路演）或从零开始。

对话技巧总结

✅ 推荐做法
  • 🥇 最佳实践：把文稿保存为 .md 文件 → 输入 /ostar-all-in-html-ppt @文档.md （可选：需要逐字稿） 回车 → Skill 自动读取内容并推荐主题和模板 → 用自然语言确认或调整。AI 会利用 Markdown 的层级结构自动排版
  • 一次性说清内容 + 受众 + 风格 + 模板
  • 用场景关键词代替具体 CSS 术语（说"小红书风"而不是"暖色 accent + 衬线字体"）
  • 说页数或时长，AI 会帮你规划结构
  • 不确定风格时，让 AI 推荐 2-3 个候选
  • 演讲场景主动说"要逐字稿"
❌ 避免做法
  • 只说"帮我做个 PPT"—— AI 只能猜，结果大概率不对
  • 在生成过程中频繁切换主题/模板，导致重复返工
  • 逐页描述每个元素的像素位置——这不是设计稿
  • 忘记说受众是谁——给 CTO 看的和给小红书读者看的内容深度完全不同

📄 单文件自包含 — 生成即交付

📄

生成的 HTML 是独立的单文件，可以复制到任何设备和路径使用

Skill 在生成 PPT 时会自动完成「内联」过程：读取仓库中的 assets/base.css、assets/runtime.js、assets/animations/animations.css、主题 CSS、模板 CSS 等外部文件，将其内容全部写入 <style> 和 <script> 标签中，最终产出一个独立的 .html 文件。

生成前：deck HTML 通过 <link> 和 <script src> 引用仓库中的共享资源，开发时修改方便
生成后：所有 CSS 和 JS 全部内联到单个 HTML 文件中，不依赖任何外部文件（仅保留 Google Fonts CDN @import，字体是二进制无法内联）
结果：你可以把这个 .html 文件 复制到任何设备、任何路径——发给同事、放 U 盘、上传网盘、用微信发送——对方双击打开即可演讲，无需安装任何软件，不需要原始仓库

🎨 36 套主题

🎨

什么是「主题」？

主题是一套 CSS 变量（Token） 的集合，定义了演示文稿的视觉基调：背景色、文字色、强调色、卡片样式、阴影、圆角、字体等 20+ 个属性。它不是布局，不是内容——它只管「长什么样」。

类比：主题就像 PowerPoint 的「配色方案 + 字体方案」
作用：换一个主题 CSS 文件 = 整个 deck 瞬间换风格，内容完全不受影响
操作：按 T 键循环切换，或在 HTML 中替换 <link id="theme-link"> 的 href。也可以用自然语言对话更换（如"帮我换成 tokyo-night 主题"）

每张卡片是一个 <iframe>，加载了该主题的真实 CSS。

📐 31 种页面布局

📐

什么是「页面布局」？

布局是单页幻灯片的 HTML 骨架——它决定了内容怎么摆：封面怎么设计、要点列表怎么排、数据图表放哪里、代码块怎么展示。每个布局是一个 <section class="slide"> 代码块，带完整的 demo 数据。

类比：布局就像 PowerPoint 的「版式 / 幻灯片母版」
与主题的关系：主题管颜色 → 布局管结构 → 两者独立组合。同一个「双栏布局」，套上 tokyo-night 是暗色技术风，套上 xiaohongshu-white 就变成小红书风
使用方式：可以用自然语言对话更换（如"帮我把封面换成双栏布局"），也可以手动操作。下面以 stat-highlight.html 为例，演示如何用 3 步把布局变成你自己 deck 的一页

什么是「动效」？

动效分两类：CSS 入场动画和Canvas 持续特效。CSS 动画是「一次性」的——元素进入幻灯片时播放一次（如淡入、上滑、弹出）；Canvas FX 是「持续性」的——在幻灯片展示期间一直运行（如粒子爆炸、烟花、矩阵雨、知识图谱）。

CSS 动画（27 个）：用 data-anim="name" 添加到任意元素，runtime.js 在每次翻到该页时自动重新触发。适合标题入场、列表逐行出现、数字跳动
Canvas FX（20 个）：用 data-fx="name" 添加到容器 div，由 fx-runtime.js 管理生命周期（进入页面启动、离开页面停止）。适合封面背景、数据可视化、庆祝页
设计原则：一页最多 1-2 种动画，混用 5 种看起来很乱。stagger-list + 一个 hero 入场 = 干净节奏

什么是「Deck 模板」？

Deck 模板是一份完整的多页幻灯片文件（通常 6-10 页），已经帮你搭配好了主题 + 布局 + 动效的组合。它不是零件，而是可以直接用的成品——打开 index.html，替换文字和图片，你的 PPT 就做好了。也可以用自然语言对话更换（如"帮我把这个模板换成 product-launch"）。

类比：Deck 模板就像 PowerPoint 的「模板 (.potx)」——封面、目录、内容页、图表页、结尾页全部就位
与布局/主题的区别：主题是配色 → 布局是单页结构 → Deck 模板是完整的多页组合，三者是层层递进的关系
来源：8 套从真实项目中提取（小红书、技术评测、架构白皮书等），7 套为通用场景脚手架（路演、发布、周报等）

什么是「演讲者模式」？

演讲者模式是一个独立弹出窗口，在你演讲时显示在演讲者自己的屏幕上（观众看不到）。它解决了演讲者最核心的焦虑——「下一页是什么？」「我讲到哪了？」「时间够吗？」「这段话怎么说？」

类比：就像 PowerPoint 的「演示者视图」——观众看到全屏幻灯片，演讲者看到备注+下一页+计时器
核心技术：每张幻灯片的 <aside class="notes"> 写逐字稿（150–300 字），按 S 弹出窗口显示大字体提词器；当前页和下一页用 iframe 像素级预览（加载同一份 HTML + CSS，颜色保证一致）
卡片可拖拽缩放：4 张磁吸卡片（当前页/下一页/提词器/计时器）的位置和大小自动保存到 localStorage，每个 deck 独立记忆
🔵 CURRENT
当前幻灯片 — 像素级 iframe 预览
🟣 NEXT
下一页 — 提前准备过渡
🟠 SPEAKER SCRIPT
逐字稿提词器 — 大字体可滚动，150–300 字/页
🟢 TIMER
计时器 + 页码 — 卡片可拖拽缩放，布局记忆
